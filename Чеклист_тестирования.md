# Чек-лист тестирования прототипа ReviewOS

## 🔐 1. Точка входа

- [ ] Открывается экран входа (главная страница)
- [ ] Видны 4 демо-доступа
- [ ] Клик по демо-доступу заполняет email и пароль
- [ ] Кнопка «Войти» переводит на правильный дашборд
- [ ] Переключатель темы ☀️/🌙 работает на экране входа

## 👤 2. Сценарий: HR-админ (hr@reviewos.ru)

### Дашборд HR
- [ ] Открывается admin-dashboard с таблицей всех сотрудников
- [ ] Видны 6 этапов цикла: Performance → Самооценка → Hard → Soft → СРВ → 1-on-1
- [ ] Показан блок «Запросы на согласование от ПО»
- [ ] Цветные индикаторы прогресса у каждого сотрудника

### Согласование Недопуска (новое!)
- [ ] Кнопка «Принять решение» открывает большое модальное окно
- [ ] Виден полный маршрут согласования (ПО ✓ → РУК ✓ → Вы)
- [ ] Видны оценки ПО по всем 5 критериям
- [ ] Видно обоснование от ПО
- [ ] Видно решение Руководителя
- [ ] Без выбора решения кнопка «Утвердить» выдаёт ошибку
- [ ] Без комментария кнопка «Утвердить» выдаёт ошибку
- [ ] При выборе «Утвердить Недопуск» появляется одно сообщение
- [ ] При выборе «Вернуть к допуску» — другое сообщение

### Навигация по админке
- [ ] Сайдбар имеет 11 пунктов и одинаков на всех страницах
- [ ] Активный пункт подсвечен на каждой странице
- [ ] **Циклы оценки** — открывается без съезда
- [ ] **Сотрудники** — открывается, видна таблица
- [ ] **Справочники** — 4 вкладки переключаются (Направления/Должности/Команды/Грейд)
- [ ] **Компетенции** — 3 вкладки (Hard/Soft/СРВ), фильтр направлений работает
- [ ] **Конструктор форм** — список шаблонов, можно открыть
- [ ] **Пользователи** — таблица + кнопки в шапке
- [ ] **Аналитика** — графики и метрики
- [ ] **Шаблоны писем** — список писем
- [ ] **Журнал писем** — список отправленных

### Справочники (новое!)
- [ ] Кнопка «+ Добавить» на каждой вкладке открывает свою форму
- [ ] Направления → форма с полями Название, Описание
- [ ] Должности → форма с Название + Направление (dropdown)
- [ ] Команды → форма с Название, Описание, Тимлид
- [ ] Грейд → форма с Название уровня, Цвет (5 вариантов), Подуровни

### Компетенции (новое!)
- [ ] Фильтр направлений — выпадающий список (без «Все»)
- [ ] При смене направления — список обновляется
- [ ] Кнопка «Импорт из другого направления» открывает модалку
- [ ] В модалке импорта — список компетенций других направлений
- [ ] Чекбоксы позволяют выбрать несколько
- [ ] Кнопка «+ Добавить компетенцию» (наверху) открывает оригинальную форму с описаниями шкалы 0-3

### Пользователи (новое!)
- [ ] Кнопка «+ Добавить вручную» в шапке открывает форму
- [ ] В форме есть «Основная роль» (4 варианта)
- [ ] Кнопка «+ Добавить дополнительную роль» добавляет новый dropdown
- [ ] Уже выбранные роли не повторяются в дополнительных
- [ ] Иконка 🗑 удаляет дополнительную роль

### Конструктор форм
- [ ] Кнопки наверху справа: Создать шаблон, Тема, Уведомления
- [ ] При редактировании Soft 360° виден блок «Логика пересчёта»
- [ ] Видна таблица маппинга 1-5 → 0-3

## 👨‍💼 3. Сценарий: Руководитель (lead@reviewos.ru)

### Дашборд руководителя
- [ ] Открывается manager-profile со статистикой
- [ ] 5 карточек: Подчинённых / Hard / СРВ / 1-on-1 / Завершено
- [ ] Видна таблица сотрудников

### Согласование Недопуска от ПО (новое!)
- [ ] Виден блок «Запрос на согласование от ПО»
- [ ] Кнопка «Рассмотреть» открывает модальное окно
- [ ] Видны оценки ПО по 5 критериям + обоснование
- [ ] Виден маршрут согласования (ПО ✓ → Вы → HR)
- [ ] Без выбора и комментария кнопка «Отправить» выдаёт ошибку
- [ ] При «Поддерживаю» — одно сообщение, при «Не согласен» — другое

### Оценка Hard Skills (модалка)
- [ ] Кнопка «Hard Skills» у Марии Ивановой открывает модальное окно
- [ ] В шапке модалки имя и должность сотрудника
- [ ] 4 компетенции, у каждой 4 карточки оценок 0/1/2/3
- [ ] Карточки одинаковой высоты (110px)
- [ ] У каждой карточки видно описание балла (Не владеет/Базово/Уверенно/Эксперт)
- [ ] Если описание длинное — внутри карточки появляется прокрутка
- [ ] Клик на карточку подсвечивает её цветом (0-красный, 3-зелёный)
- [ ] Кнопки Отмена / Черновик / Сохранить работают

### Оценка СРВ (модалка)
- [ ] Кнопка «Оценить СРВ» у Ольги Смирновой открывает свою модалку
- [ ] Имя и должность Ольги в шапке
- [ ] 4 компетенции (Самостоятельность, Ответственность, Коммуникация, Командная работа)
- [ ] Описания баллов видны и работают как в Hard

### Также как сотрудник
- [ ] Виден блок «Мои задачи» — заполнить самооценку, оценить коллег
- [ ] Кнопки ведут на нужные формы

## 📦 4. Сценарий: Продакт-оунер (po@reviewos.ru)

### Дашборд ПО
- [ ] Открывается po-profile
- [ ] 4 карточки: Сотрудников / Оценено / Ожидают / На согласовании
- [ ] Видна таблица 5 сотрудников

### Оценка Performance (модалка)
- [ ] Клик «Оценить» у любого сотрудника открывает модалку
- [ ] В шапке имя и должность
- [ ] 5 критериев — у каждого только название, описание и поле комментария (без Допуск/Недопуск!)
- [ ] Внизу один блок «Итоговое решение»
- [ ] При выборе «Допуск» сохраняется как обычно
- [ ] При выборе «Недопуск» появляется блок эскалации

### Эскалация Недопуска (новое!)
- [ ] При «Недопуске» появляется жёлтый блок «Решение требует согласования»
- [ ] Видно объяснение что отправляется в РУК и HR
- [ ] Видно поле «Обоснование недопуска» (обязательное)
- [ ] Видно визуальный маршрут (ПО → РУК → HR)
- [ ] Кнопка «Сохранить» меняется на «Отправить на согласование»
- [ ] Без обоснования — ошибка
- [ ] При отправке — сообщение об эскалации
- [ ] У Татьяны Беловой статус «На рассмотрении»
